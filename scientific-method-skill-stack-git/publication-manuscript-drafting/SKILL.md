# Publication Manuscript Drafting

Turn a completed scientific workflow into a manuscript-ready draft.

This is not a general reporting skill.
This is not a lab-update skill.
This is not a discussion-only skill.

This skill is for publication-shaped writing.

## Relationship to scientific-reporting-v1

`scientific-reporting-v1` is the broad communication layer for reports, updates, posters, talks, and narrative scientific outputs across the pipeline.

`publication-manuscript-drafting` is narrower.

It is specifically for:
- manuscript architecture
- section logic
- journal-calibrated claim flow
- figure/table orchestration
- supplement-aware assembly
- title and abstract refinement at manuscript level

## Role in the pipeline

This is the final publication-assembly skill in the scientific-method stack.

Its central question is:
- **How should this completed scientific workflow be assembled into a manuscript-shaped argument, at the appropriate stage of drafting?**

This skill converts the full scientific workflow into publication-ready manuscript architecture and prose.

It is not:
- the analysis skill
- the interpretation skill
- the conclusion skill
- the general reporting skill

It is the final manuscript-packaging layer.

If upstream scientific logic is incomplete, do not fake a finished manuscript. State what is missing.

## Boundary

This skill should stop after producing a Manuscript Draft Package.

It may:
- assemble the manuscript argument
- choose manuscript type and scope
- decide what belongs in main text versus supplement
- produce title options
- produce abstract, outline, or full draft text depending on stage
- organize figures, tables, legends, and supplement structure
- identify reviewer vulnerabilities
- state what is still missing before the next drafting stage

It should not:
- invent missing data
- silently fix scientific logic gaps with prettier prose
- overstate conclusions beyond the `Conclusion Block`
- perform new science unless clearly marked as missing-support work
- hide upstream uncertainty or indeterminacy

Assemble and sharpen, yes.
Create unsupported science out of thin air, no.

The manuscript stage may need to recover:
- the original motivating phenomenon
- why the question mattered
- what competing hypotheses were considered
- why the chosen design was used
- what was planned versus what actually happened
- what remained unresolved at each stage

The skill may use upstream artifacts to recover rationale, methods logic, and narrative scaffolding.

It must not use upstream access to smuggle in claims that outrun the `Conclusion Block`.

## Draft stages

This skill must explicitly operate in one of three stages:

### 1. Outline stage
Use when:
- the science is sufficiently shaped to define the paper architecture
- but full prose drafting is premature or inefficient

This stage should produce:
- manuscript identity
- claim scope
- argument map
- section-by-section outline
- figure/table plan
- main text versus supplement plan
- missing-support list

### 2. First-draft stage
Use when:
- the argument is stable enough for prose drafting
- figures and tables are sufficiently defined
- known weaknesses can remain visible and labeled

This stage should produce:
- manuscript outline plus
- title candidates
- abstract draft
- introduction draft
- methods draft
- results draft
- discussion draft
- figure legend draft
- table framing draft
- explicit unresolved issues

### 3. Final-draft stage
Use when:
- figures and tables are stable
- claims are calibrated
- conclusion is stable
- reviewer-facing vulnerabilities have been considered

This stage should produce:
- a near-submission manuscript package
- tightened prose
- aligned title, abstract, results, discussion, and conclusion
- refined figure/table placement
- explicit supplement logic
- remaining minor cleanup list only

This skill must not silently jump from weak evidence to polished final-draft confidence.

## Draft readiness gate

Before drafting, the skill must state:

# Draft readiness gate

- evidence chain complete: yes / partially / no
- figure support sufficient: yes / partially / no
- table support sufficient: yes / partially / no
- conclusion calibrated: yes / partially / no
- major scientific gap still open: yes / no
- major methods ambiguity still open: yes / no
- manuscript stage justified:
  - outline only
  - first draft justified
  - final draft justified
  - not yet manuscript-ready

If the project is not honestly ready for the requested stage, say so plainly.

## Purpose

This skill assembles a manuscript-shaped object with strong internal logic.

It should answer:
- what is the paper’s central claim?
- what belongs in the main text?
- what belongs in supplement?
- what is the minimal sufficient figure set?
- what does each section need to do?
- where is the manuscript vulnerable to reviewer criticism?
- what drafting stage is actually justified?

## Governing principles

### 1. Manuscripts are arguments, not containers

A manuscript is not just a report with headings.

It is a structured scientific argument.

Every section should advance that argument.

### 2. The title and abstract must carry real weight

The title should make the manuscript legible and interesting.
The abstract should stand alone and communicate:
- problem
- approach
- result
- interpretation
- scope

### 3. The introduction must create necessity

A strong introduction should:
1. define the big problem
2. specify the gap
3. explain why the gap matters
4. show why existing work is insufficient
5. state what this study does

Do not let introductions become literature dumps.

### 4. The results section is the spine

The results should be organized as a sequence of mini-arguments.

Each subsection should answer:
- what question was being addressed?
- what was done?
- what was found?
- what did that establish?

### 5. The discussion should interpret, not repeat

The discussion should:
- answer the paper’s main question
- explain why the findings matter
- situate them relative to prior work
- address limitations
- state the next conceptual step

### 6. Main text vs supplement must be deliberate

Keep the main text focused on the core argument.

Move to supplement when material is:
- important but secondary
- confirmatory but not central
- methodological detail too large for the main text
- sensitivity analyses
- extended QC
- supporting figure sets

### 7. Figures and tables are part of the argument

Every figure should answer a real question.
Every table should reduce reader effort.

Do not overload the main text with decorative or redundant visual material.

### 8. Reviewer vulnerability should be visible before submission

Explicitly look for:
- overclaiming
- confound vulnerability
- missing controls
- pseudoreplication risk
- under-described methods
- unsupported generalization
- mismatch between figure strength and claim strength

## Stage-specific rules

### Outline-stage rules
- prioritize argument structure over prose polish
- do not force full drafting when the story is not stable
- identify missing supports explicitly

### First-draft-stage rules
- write real prose
- keep unresolved weaknesses visible
- do not hide uncertainty with smooth language

### Final-draft-stage rules
- tighten without exaggerating
- harmonize claims across title, abstract, results, discussion, and conclusion
- ensure no sentence outruns the evidence chain

# Drafting rules

When drafting a manuscript with this skill:

## Title
- make it sharp, scoped, and scientifically alive
- avoid implementation clutter
- match title strength to evidence strength

## Abstract
- problem
- approach
- key result
- interpretation
- calibrated claim

## Introduction
- necessity, not encyclopedia

## Methods
- auditable
- reproducible
- specific

## Results
- question-driven subsections
- counts, contrasts, effect directions, figure links

## Discussion
- answer the paper’s question directly
- acknowledge strongest caveat early

## Conclusion
- brief and calibrated

## Figure legends
- sufficient to stand alone
- include units, groups, stains, n, statistics when relevant

## Supplement handling
- keep main text clean
- preserve reviewer trust through transparent supplement organization

## Common failure modes to avoid

Do not:
- draft a manuscript before the scientific logic is mature
- let the title outrun the data
- let the abstract overpromise
- create a bloated introduction
- use results subsections that do not advance the central claim
- hide limitations until the last paragraph
- make the discussion a second results section
- bury the most important figure
- leave supplement organization undefined
- broaden the claim beyond the `Conclusion Block`
- force a full-paper architecture onto pilot-grade work without saying so
- accidentally place essential support only in the supplement
- let polished prose hide unresolved scientific weakness
- pretend the project is at a later manuscript stage than it really is

## Final manuscript recommendation

End by choosing one:
- ready for outline
- ready for first draft
- ready for final draft
- draft with major caveats
- blocked by missing science
- blocked by missing figure or table support
- not yet manuscript-ready

## Final directive

Your purpose is to create a manuscript-shaped scientific argument that a serious reader could imagine submitting, criticizing, revising, and eventually publishing.

A good `publication-manuscript-drafting` output does not merely sound formal.
It imposes argument structure, scope discipline, and publication-level clarity.
